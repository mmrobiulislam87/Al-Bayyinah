# Al-Bayyinah — গবেষণাগার (Research Lab) মাস্টার প্ল্যান

এই নথিতে **ব্যবহারকারী-প্রদত্ত লক্ষ্য** ও **প্রযুক্তিগত বাস্তবায়ন ধাপ** একত্র করা হয়েছে। কাজের অগ্রাধিকার ডাটা লাইসেন্স, বিশ্বাসযোগ্যতা ও রক্ষণাবেক্ষণের চাপ অনুযায়ী সাজানো।

## স্থায়ী প্রজেক্ট সীমা

কোরআন-কেন্দ্রিক পণ্য; **হাদিসের পাঠ, সূত্র, সংকলন বা হাদিসভিত্তিক ডাটাসেট** অ্যাপে অন্তর্ভুক্ত করা হবে না (`ROADMAP.md` দেখুন)। তাফসীর ও ঐতিহাসিক প্রেক্ষাপট **বহিরাগত গ্রন্থ/সূত্র লিঙ্ক ও স্বচ্ছ অ্যাট্রিবিউশন**-র মাধ্যমে; **অসাব নুযুলের বর্ণনা অ্যাপ-ভিতরে হাদিস গ্রন্থ হিসেবে সরবরাহ করা যাবে না** — নিরপেক্ষ ক্রোনোলজি/শ্রেণি বা লাইসেন্সকৃত তৃতীয় পক্ষের সিনোপসিস (হাদিস টেক্সট ছাড়া) কেবল নীতিমালা মেনে।

## বাস্তবায়ন পর্যায় (সঞ্চারপথ)

1. **A — ভেক্টর সার্চ + কorpus শক্তিশালীকরণ** (FTS-পরবর্তী ধাপ)
2. **B — ওয়ার্কস্পেস v0 (নোট, এক্সপোর্ট স্টাব)** ব্যবহারকারী মূল্য দ্রুত
3. **C — অন্টোলোজি v0 (সীমিত নোড/এজ) + গ্রাফ UI**
4. **D — মরফো+সিনট্যাক্স ইমপোর্ট** → ই‘রাব UI
5. **E — কিরাআত টগল** (ডাটা নিশ্চিত হলে)
6. **F — মুনাসাবাহ / চiasmus / তাফসীর প্যারালাল** (কন্টেন্ট পাইপলাইনের সাথে)

কোডবেসে হাব: **`/research/lab`** — মডিউল ট্যাব ও `web/lib/researchLab/modules.ts`-এ স্থিতি ট্র্যাকিং।

### চালু বা ভূমিকা-স্তর (`v0`)

| পথ / মডিউল | বিষয়বস্তু |
|-------------|-------------|
| `GET /api/research/meaning-search` | `_searchText` অনুবাদ ব্লবে টোকেন মিল (ভেক্টর নয়) |
| সূরা পাতা → «গবেষক নোট» | `AyahBlock` লিঙ্ক: `/research/lab?m=workspace&v=সূরা:আয়াত` |
| `supabase/migrations/20260506140000_ayah_embedding_pgvector.sql` | `pgvector` + `ayah_embedding_chunks` স্ক্যাফোল্ড (ইটিএল পূরণ ও ইনডেক্স পরে) |
| গবেষণাগার | অন্টোলজি সিড, মুনাসাবাহ নমুনা, ক্রোনোলজি টেবিল, নোট+এক্সপোর্ট, তাফসীর হাব লিঙ্ক |
| `/research/tafsir` | আয়াত লোড + বহিরাগত সম্পদ তালিকা |

## একাডেমিক স্বচ্ছতা

প্রতিটি ডাটাসেট ও গ্রন্থের **লাইসեն্স ও উদ্ধৃতি**; স্ক্র্যাপিং এড়িয়ে আনুষ্ঠানিক সূত্র। ফলাফল UI-তে «এটি পাঠগত/তাত্ত্বিক সূচক» ইত্যাদি যেখানে প্রযোজ্য।
